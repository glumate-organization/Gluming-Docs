# 랜딩 페이지 설계

## 기본 섹션 흐름 (마케팅 랜딩)

1. **히어로** — 한 문장 가치제안 + 핵심 CTA(앱 다운로드/사전등록). 글루밍 thesis를 1초에 전달: "행동 전에 혈당을 미리 시뮬레이션"
2. **문제 정의** — CGM 데이터가 행동으로 안 이어지는 페인포인트 (사후 분석의 한계)
3. **해결/기능** — 사전 시뮬레이션(식사·운동), 멀티호라이즌 예측, 개인화 추천을 3~4 블록으로
4. **타깃 적합성** — 세그먼트별 "이런 분께" (임신성 당뇨 / 검진 트리거 / 식단관리). 웰니스 톤 유지
5. **신뢰 요소** — (있다면) 연구 배경, 진행 단계. 과장 금지
6. **최종 CTA** — 동일 액션 반복
7. **푸터** — 회사 정보, 의료 비대체 고지, 개인정보/약관 링크

> 게시글(혈당/당뇨 콘텐츠)은 **지금 범위 아님**. 향후 Content Collections로 `/blog` 추가 시 네비에 자리만 비워둘 수 있음.

## 전환 설계

- CTA는 페이지당 **하나의 주요 액션**으로 통일 (혼란 방지)
- 히어로 CTA는 스크롤 없이 보이게 (above the fold)
- 버튼 카피는 행동 동사 + 가치: "사전등록하고 먼저 써보기"
- 폼은 최소 필드. 정적 사이트이므로 제출은 외부 폼 서비스 대신 메일링/구글폼 링크 또는 향후 서버 연동으로 (자가완결 원칙상 외부 스크립트 임베드는 신중히)

## 카피 위계

- H1 1개(히어로), 섹션마다 H2, 그 안에 H3
- 한 섹션 = 한 메시지. 스캔 가능하게 짧게
- 웰니스 표현 규칙은 `wellness-content` 스킬 준수

## 접근성 & 성능 (정적 사이트 강점 살리기)

- 모든 `<img>`에 `alt`, 장식 이미지는 `alt=""`
- 색 대비 WCAG AA, 키보드 포커스 가시
- 이미지 `width`/`height` 지정해 CLS 방지
- 자가완결 자산(인라인/번들)이라 LCP 유리 — 이 강점을 헤드라인 이미지에 활용
- JS 최소화 (Astro 아일랜드: 인터랙션 필요한 곳만 client:* 지시어)
- Lighthouse 90+ 목표

## 디자인 토큰

- 색/타이포/간격은 `_guide/brand.md`에서 가져옴. 임의 색상값 흩뿌리지 말고 CSS 변수로 관리.
